# CRUD-приложение для склада товаров

Небольшое приложение для управления товарами на складе, написанное на Java 17 с использованием Spring, Hibernate, 
PostgresSQL

## Запуск

Запуск приложения можно осуществить двумя способами

### Запуск ручками

Для запуска ручками из под dev

```shell
mvn spring-boot:run
```

Приложение запустится на порте 9999, используя in-memory базу данных 

---
Для запуска из под prod

```shell
# Задаем переменные для подключения к PostgreSQL
set POSTGRES_USER = username
set POSTGRES_PASSWORD = password
set POSTGRES_HOST = database
set POSTGRES_PORT = 5432
set POSTGRES_DB = stockhouse_db
# Запускаем приложение
mvn -Pprod spring-boot:run
```
Приложение запустится на порте 8080, используя PostgreSQL в качестве базы данных

---

### Запуск через Docker-compose

Для запуска сервиса

```shell
docker-compose run --rm backend
```

К приложению можно будет получить доступ по 9999 порту

---

Для его остановки

```shell
docker-compose down --remove-orphans
```

---

Для удаления всех данных и очистки всех образов сервиса

```shell
docker-compose down --rmi all
```

## Профили приложения

Для удобства запуска в различных окружениях (локальный запуск или запуск в продакшене)
есть два профиля

#### Dev профиль
В Maven выбирается как `-Pdev`, профиль по умолчанию

Особенности:
- Использование in-memory базы данных H2 заместо PostgreSQL
- Не требует никаких настроек, работает из коробки
- Удобен для локального запуска
- Запуск на 9999 порте

#### Prod профиль
В Maven выбирается как `-Pprod`, используется в докере

Особенности:
- Использование PostgreSQL
- Требует настройки переменных окружения для подключения с PostgreSQL
- Удобен для запуска в докере
- Запуск на 8080 порте

## Swagger и OpenAPI

В приложении реализована поддержка спецификации OpenAPIv3, а также присутствует Swagger-ui

----

#### Адрес по которому лежат OpenAPI docs

`/swagger-ui/index.html`

----

#### Адрес по которому можно получить доступ к Swagger UI

Для доков в формате `.json`

- `/api-docs`

Для доков в формате `.yaml`

- `/api-docs.yaml`
